# Picora — 技术设计文档

## 项目概述

为 80+ 岁老人设计的本地照片管理器，核心理念是"时间的长河"——所有照片按拍摄时间排列成一条连续的时间线，不暴露文件夹/子文件夹的概念。

**目标用户：** 老年人，只需要浏览和删除照片，不需要编辑功能。

**核心体验：** 打开软件 → 看到所有照片的时间线 → 滚动浏览 → 双击放大看 → 不想要的删掉（安全删除到回收站）。

---

## 技术选型

| 类别 | 选择 | 理由 |
|------|------|------|
| 桌面框架 | Electron | 跨平台、生态成熟、开发效率高 |
| 前端框架 | React 18+ | 团队熟悉、生态丰富 |
| 虚拟滚动 | react-window | 轻量、性能好、满足列表虚拟化需求 |
| 图片处理 | sharp | Node.js 生态最强的图片处理库，生成缩略图速度快 |
| EXIF 读取 | exifr | 速度快，只读文件头，支持批量读取 |
| 构建打包 | electron-builder | 支持 Windows 打包，社区成熟 |
| 包管理 | npm | 标准方案 |

### IPC 通信设计

通过 `contextBridge` 暴露安全的 API 给渲染进程，不直接暴露 Node.js 能力。

```typescript
// preload.ts 中暴露的 API
interface PicoraAPI {
  // 配置
  getConfig(): Promise<AppConfig>
  saveConfig(config: AppConfig): Promise<void>

  // 扫描
  startScan(): Promise<void>
  onScanProgress(callback: (progress: ScanProgress) => void): void

  // 照片数据
  getPhotos(page: number, pageSize: number): Promise<PhotoPage>
  getPhotoCount(): Promise<number>

  // 缩略图
  getThumbnail(photoId: string): Promise<string>  // 返回 base64 或文件路径

  // 查看原图
  getFullImage(photoId: string): Promise<string>  // 返回文件路径

  // 删除
  deletePhoto(photoId: string): Promise<boolean>

  // 选择文件夹
  selectFolder(): Promise<string | null>

  // 事件监听
  onPhotosUpdated(callback: () => void): void
}
```

## 数据模型

### 照片索引 (photos.json)

```json
{
  "version": 1,
  "lastScan": "2026-06-16T22:00:00.000Z",
  "folders": ["D:\\照片", "E:\\手机备份"],
  "photos": [
    {
      "id": "a1b2c3d4",
      "path": "D:\\照片\\2024\\01\\IMG_0001.jpg",
      "dateTaken": "2024-01-15T14:30:00.000Z",
      "width": 4032,
      "height": 3024,
      "thumbGenerated": true,
      "thumbPath": "thumbnails/a1b2c3d4.webp"
    }
  ]
}
```

**字段说明：**
- `id`：基于文件路径的短哈希（8 位），用于缩略图命名和 API 调用
- `dateTaken`：优先取 EXIF DateTimeOriginal，没有则取文件修改时间
- `thumbGenerated`：标记缩略图是否已生成，支持渐进式生成
- `thumbPath`：相对于 userData 目录的缩略图路径

### 应用配置 (config.json)

```json
{
  "folders": ["D:\\照片", "E:\\手机备份"],
  "thumbnailSize": 300,
  "scanOnStartup": true
}
```

### 存储位置

所有应用数据存储在 `app.getPath('userData')` 目录下：

```
%APPDATA%/picora-app/
├── config.json          # 应用配置
├── photos.json          # 照片索引
└── thumbnails/          # 缩略图缓存
    ├── a1b2c3d4.webp
    ├── e5f6g7h8.webp
    └── ...
```

缩略图使用 webp 格式（体积小、质量好，300px 宽大约 10-20KB）。

## 核心流程

### 1. 首次启动流程

```
启动应用
  → 检测 config.json 是否存在
  → 不存在 → 显示设置页面
  → 用户选择照片文件夹（可多选）
  → 保存配置
  → 开始扫描
```

### 2. 扫描流程

```
扫描开始
  → 递归遍历所有配置的文件夹，收集图片文件路径
      支持格式：jpg, jpeg, png, heic, heif, webp, bmp, tiff
  → 快速读取 EXIF 日期（只读文件头，exifr.parse 的 tiff.ifd0 和 exif 段）
  → 生成 photos.json 索引（此时 thumbGenerated = false）
  → UI 立即显示时间线骨架（有日期但无缩略图）
  → 后台开始生成缩略图（从最新的照片开始，向前推进）
  → 每生成一批（50 张），通知 UI 刷新
  → 全部完成 → UI 完整显示
```

### 3. 日常启动流程

```
启动应用
  → 加载 photos.json
  → 校验每条记录的文件路径是否存在（快速 stat 检查）
  → 清理不存在的记录
  → 如果配置了 scanOnStartup，后台扫描新照片
  → 显示时间线
```

### 4. 浏览流程

```
时间线页面
  → react-window 虚拟滚动，只渲染可见区域的缩略图
  → 按 dateTaken 降序排列（最新的在最上面）
  → 年月分隔标题（如 "2024年1月"）
  → 缩略图以网格形式排列（每行 4-6 张，响应式）
  → 滚到顶部/底部时按需加载

双击照片
  → 进入全屏看图模式
  → 显示原图（或大图，限制在屏幕分辨率内）
  → 左右箭头键 / 屏幕左右按钮翻页
  → Esc 键 / 关闭按钮返回时间线
  → 底部显示拍摄日期
```

### 5. 删除流程

```
用户点击删除按钮（看图模式下或右键菜单）
  → 弹出确认对话框："确定删除这张照片吗？（可在回收站找回）"
  → 确认 → shell.trashItem(originalPath)
  → 删除缩略图文件
  → 从 photos.json 中移除记录
  → UI 刷新，照片从时间线消失
  → 如果在看图模式，自动跳到下一张
```

**布局说明：**
- 左侧固定宽度（约 180-220px），显示年-月树形结构，可折叠展开
- 右侧占满剩余空间，显示所有照片的缩略图网格（虚拟滚动）
- 左右两侧之间可以有一条可拖动的分隔线（可选）

**时间树（左侧面板）：**
- 年份为一级节点，可折叠；月份为二级节点
- 时间树是**导航游标**，不是过滤器——点击某个月份，右侧滚动到该月照片的位置，但右侧始终显示所有照片的连续时间线
- 右侧滚动时，左侧自动高亮当前可视区域对应的年月（联动效果）
- 当前月份左侧显示一个标记（如 `←` 或高亮背景），让老人家知道"我看到的是什么时候的照片"
- 每个月份后面可以显示照片数量（如 `3月 (128)`）

**缩略图区域（右侧面板）：**
- react-window 虚拟滚动，只渲染可见区域的缩略图
- 按 dateTaken 降序排列（最新的在最上面）
- 缩略图以网格形式排列（每行 4-6 张，响应式）
- 月份之间有一条淡淡的分隔线 + 小标题（如 "2024年3月"），辅助视觉分区
- 滚到顶部/底部时按需加载
```

- 时间分隔标题吸顶显示，滚动时始终能看到当前所在月份
- 首次扫描时显示进度条和已完成数量

## UI/UX 设计原则（面向老年人）

1. **字体大**：基础字号 16px 以上，标题更大
2. **按钮大**：可点击区域至少 44x44px
3. **高对比度**：深色文字 + 浅色背景（时间线），深色背景 + 亮色照片（看图）
4. **少选项**：每个页面操作不超过 3 个
5. **明确反馈**：删除有确认弹窗，扫描有进度条，操作完成有视觉反馈
6. **无学习成本**：不出现"文件夹"、"索引"、"缓存"等技术词汇
7. **容错设计**：删除可恢复，误操作无严重后果

## 打包与分发

### 开发环境

```bash
# Mac 上开发
npm install
npm run dev          # 同时启动 Electron + React dev server
```

### 打包 Windows 版本

```bash
# 在父亲 Windows 电脑上
git clone <repo>
cd picora
npm install
npm run build        # electron-builder 打包成 Windows 安装包
```

输出：`dist/Picora Setup x.x.x.exe`

### electron-builder 配置

```yaml
appId: com.picora.app
productName: Picora
directories:
  output: dist
win:
  target: nsis       # 生成 Windows 安装向导
  icon: resources/icon.png
nsis:
  oneClick: true     # 一键安装，不让老人选路径
  perMachine: true
```

## 技术风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 几万张照片首次扫描慢 | 用户等待时间长 | EXIF 只读文件头 + 后台异步 + 进度反馈 |
| sharp 在 Windows 上安装问题 | 打包失败 | sharp 提供预编译二进制，electron-rebuild 处理 |
| 大文件 HEIC 格式 | iPhone 照片无法显示 | sharp 原生支持 HEIC 解码 |
| 内存占用 | 几万条索引 + 缩略图 | 虚拟滚动 + 分页加载 + 缩略图 webp 压缩 |
